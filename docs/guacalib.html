guacalib

Guacamole Database Management Library.

This library provides programmatic access to Apache Guacamole database operations, enabling automated management of users, user groups, connections, and connection groups through direct MySQL database access. It serves as both a Python library and the foundation for the guacaman CLI tool.

The library focuses on security, reliability, and comprehensive functionality for managing Guacamole deployments programmatically. It supports all major Guacamole features including user management, permission control, connection configuration, and hierarchical organization.

Key Features:

  • Complete user and user group management
  • Connection and connection group operations
  • Fine-grained permission control
  • Hierarchical organization support
  • Transaction-based operations
  • Comprehensive error handling and validation
  • MySQL database connectivity with connection pooling

Example: Basic usage for user management:

>>> from guacalib import GuacamoleDB
>>>
>>> # Initialize database connection
>>> with GuacamoleDB() as db:
>>>     # Create a new user
>>>     db.create_user("testuser", "securepassword")
>>>
>>>     # Create a VNC connection
>>>     conn_id = db.create_connection(
>>>         "vnc", "desktop", "192.168.1.100", "5901", "vncpassword"
>>>     )
>>>
>>>     # Grant user access to connection
>>>     db.grant_connection_permission("testuser", "USER", conn_id)

Example: Connection group management:

>>> from guacalib import GuacamoleDB
>>>
>>> with GuacamoleDB() as db:
>>>     # Create a connection group
>>>     group_id = db.create_connection_group("Desktops", "ORGANIZATIONAL")
>>>
>>>     # Add connection to group
>>>     db.create_connection(
>>>         "vnc", "desktop-pc", "192.168.1.101", "5901", "password",
>>>         parent_group_id=group_id
>>>     )
>>>
>>>     # Grant group access to user group
>>>     db.grant_connection_group_permission_to_user(
>>>         "IT Staff", "USER_GROUP", group_id
>>>     )

API Overview: The main entry point is the GuacamoleDB class, which provides methods for:

- **User Management**: create_user(), delete_existing_user(), modify_user()
- **User Groups**: create_usergroup(), add_user_to_usergroup()
- **Connections**: create_connection(), modify_connection(), delete_existing_connection()
- **Connection Groups**: create_connection_group(), modify_connection_group_parent()
- **Permissions**: grant_connection_permission(), revoke_connection_permission()
- **Data Listing**: list_users(), list_connections_with_conngroups_and_parents()

The GuacamoleDB class implements the context manager protocol for safe
database connection handling and supports MySQL connection parameters via
configuration files or direct parameter passing.

Security: - All SQL queries use parameterized statements to prevent injection - Database credentials stored in separate configuration files - Comprehensive input validation and error handling - Transaction support for atomic operations

Logging (Phase 4): The library provides comprehensive logging support for downstream applications:

>>> from guacalib import setup_logging, get_logger
>>>
>>> # Configure logging (optional - defaults to WARNING level)
>>> setup_logging(debug=True)  # Enable debug logging
>>>
>>> # Get a module-specific logger
>>> logger = get_logger('myapp')
>>> logger.info("Application started")
>>>
>>> # Environment variables can override logging configuration:
>>> # export GUACALIB_LOG_LEVEL=DEBUG
>>> # export GUACALIB_LOG_FORMAT='%(asctime)s [%(levelname)s] %(name)s: %(message)s'
>>> setup_logging()

Important: setup_logging() is never called automatically on import to respect
host application logging configuration. Applications must call it explicitly
if they want guacalib logging configured.

Logging Features:
- Environment variable support (GUACALIB_LOG_LEVEL, GUACALIB_LOG_FORMAT)
- Idempotent configuration - safe to call multiple times
- Uses stderr to preserve stdout for application data

 170    def debug_print(self, *args: Any, **kwargs: Any) -> None:
 171        """Print debug messages if debug mode is enabled with credential scrubbing.
 172
 173        This method provides backward compatibility by using logging when configured
 174        and falling back to print statements when logging is not available. All
 175        messages are scrubbed for sensitive credentials before output.
 176
 177        Args:
 178            *args: Arguments to pass to print function or logger. All arguments
 179                  will be converted to strings and scrubbed for credentials.
 180            **kwargs: Keyword arguments to pass to print function or logger.
 181
 182        Note:
 183            This method automatically scrubs all arguments for sensitive
 184            credentials before output to prevent accidental exposure in logs
 185            or debug output. It maintains the original debug functionality
 186            while enhancing security.
 187
 188        Example:
 189            >>> self.debug_print("Connecting with password=secret123")
 190            [DEBUG] Connecting with password=[REDACTED]
 191        """
 192        if self.debug:
 193            # Convert all arguments to strings and scrub credentials
 194            scrubbed_args = []
 195            for arg in args:
 196                if isinstance(arg, str):
 197                    scrubbed_args.append(self._scrub_credentials(arg))
 198                else:
 199                    # For non-string arguments, convert to string then scrub
 200                    scrubbed_args.append(self._scrub_credentials(str(arg)))
 201
 202            # Try to use logging if it's configured, otherwise fall back to print
 203            # Check if logging has been configured by checking if the logger has handlers
 204            if hasattr(self.logger, "handlers") and self.logger.handlers:
 205                self.logger.debug(" ".join(scrubbed_args), **kwargs)
 206            else:
 207                # Fallback to original print behavior
 208                print("[DEBUG]", *scrubbed_args, **kwargs)

 218    def __exit__(
 219        self,
 220        exc_type: Optional[Type[BaseException]],
 221        exc_value: Optional[BaseException],
 222        traceback: Optional[Any],
 223    ) -> None:
 224        """Exit the runtime context and handle database connection cleanup.
 225
 226        Commits transactions if no exception occurred, rolls back if there was
 227        an exception, and closes database connections.
 228
 229        Args:
 230            exc_type: Exception type if an exception occurred, None otherwise.
 231            exc_value: Exception value if an exception occurred, None otherwise.
 232            traceback: Exception traceback if an exception occurred, None otherwise.
 233        """
 234        if self.cursor:
 235            self.cursor.close()
 236        if self.conn:
 237            try:
 238                # Always commit unless there was an exception
 239                if exc_type is None:
 240                    self.conn.commit()
 241                    self.debug_print("Transaction committed successfully")
 242                elif exc_type is SystemExit:
 243                    # SystemExit (from sys.exit()) is used for normal CLI termination
 244                    # Commit the transaction before exiting
 245                    self.conn.commit()
 246                    self.debug_print(
 247                        "Transaction committed successfully before SystemExit"
 248                    )
 249                else:
 250                    self.conn.rollback()
 251                    self.logger.warning(
 252                        f"Transaction rolled back due to {exc_type.__name__}"
 253                    )
 254            finally:
 255                if self.debug:
 256                    self.debug_print("Closing database connection")
 257                self.conn.close()

 379    def connect_db(self) -> MySQLConnection:
 380        """Establish MySQL database connection using loaded configuration.
 381
 382        Creates a MySQL connection using the configuration loaded by read_config().
 383        Sets UTF8MB4 charset and collation for proper Unicode support.
 384
 385        Returns:
 386            MySQLConnection object for database operations.
 387
 388        Raises:
 389            SystemExit: If database connection fails.
 390            mysql.connector.Error: For various MySQL connection errors.
 391

 393            Uses UTF8MB4 charset to support full Unicode including emoji and
 394            special characters. Connection parameters are loaded from the
 395            configuration file during initialization.
 396        """
 397        try:
 398            conn = mysql.connector.connect(
 399                **self.db_config, charset="utf8mb4", collation="utf8mb4_general_ci"
 400            )
 401            self.logger.info(
 402                f"Database connection established to {self.db_config.get('host', 'unknown')}"
 403            )
 404            return conn
 405        except mysql.connector.Error as e:
 406            self.logger.error(
 407                f"Database connection failed: {self._scrub_credentials(str(e))}"
